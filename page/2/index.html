Linux 学习之路

Posted on 2019-03-17

Linux 基础

关于 Linux 的基本知识就不记录了。

记录一些容易忘记的比较难记忆的知识。

Linux 启动过程

运行 init

init 进程是所有进程的起点，首先读取 配置文件 /etc/inittab

运行级别：

Linux 系统有 7 个运行级别：

  • 0：系统停机状态，系统默认运行级别不能设置为 0，否则无法正常启动

  • 1：单用户工作状态 ––- root权限，用于系统维护，禁止远程登陆

  • 2：多用户状态 ––- 无 NFS -> Network File System

    NFS（Network File System）即网络文件系统，它允许网络中的计算机之间通过TCP/IP网络共享资源。在NFS的应用中，本地NFS的客户端应用可以透明地读写位于远端NFS服务器上的文件，就像访问本地文件一样。

  • 3：完全多用户状态 (有NFS)，登陆后进入控制台命令行模式

  • 4：未使用，保留

  • 5：X11看控制台，登陆后进入 图形化界面

  • 6：系统正常关闭并重启，默认运行级别不能设置为6

Linux 各类文件夹

  • /bin : Binary，目录中存放着最经常使用的命令。
  • /boot : 启动的核心文件 ––- 包含一些连接文件及镜像文件
  • /dev : Device，存放 Linux 的外部设备，Linux 中访问设备的方式和访问文件的方式是相同的
  • /etc : 存放所有的系统管理所需要的配置文件和子目录
  • /home : 用户的主目录，每个用户都有自己的目录，一般目录名是以用户的账户命名的。
  • /lib : 存放系统最基本的动态连接共享库，作用类似于 Windows 中的 DLL 文件。基本上所有的应用程序都要用到这些共享库
  • /lost+found : 此目录一般是空的，如果系统非法关机之后，这里就存放了一些文件
  • /media : Linux 会自动识别一些设备，例如 U 盘、光驱等等，识别之后会挂在到这个目录下
  • /mnt : 系统提供该目录是为了让用户临时挂载别的文件系统
  • /opt : 给主机额外安装软件的目录
  • /proc : 一个虚拟目录，是系统内存的映射，可以通过直接访问这个目录来获取系统信息，这个目录的内容不在硬盘上！在内存里
  • /root : 该目录为系统管理员目录
  • /sbin: s->Super User，存放系统管理员的系统管理程序
  • /selinux : Redhat/CentOS 特有目录，Selinux 是一个安全机制，类似于 Windows 防火墙，此目录即存放相关文件的
  • /srv : 存放一些服务启动后需要提取的数据
  • /sys : 当一个内核对象被创建的时候，对应的文件和目录也在内核对象子系统中被创建。
  • /tmp : 这个目录用来存放一些临时文件
  • /usr : 用户的很多应用程序和文件放在这个目录下，类似于 Windows 下的 Program files 目录
  • /usr/bin : 系统用户使用的应用程序
  • /usr/sbin : 超级用户使用的比较高级的管理程序和系统守护程序
  • /usr/src : 内核源代码默认的放置目录
  • /var : 存放在不断扩充的东西，那些经常被修改的目录放在这个目录下，包括各种日志文件

Linux 文件基本属性

Linux 是典型的多用户系统，不同用户拥有不同权限。为了保护系统的安全性，Linux 系统对不同的用户访问同一文件 (包括目录文件) 的权限做了不同的规定。

文件属性

利用 ll 或 ls -l 来显示一个文件的属性及文件所属的用户和组，如：

bin 文件的第一个属性 dr-xr-xr-x :

第一个字符含义：

  • [d] : 目录
  • [-] : 文件
  • [/] : 链接文档 link file
  • [b] : 装置文件里面可供储存的接口设备 (可随机存取装置)
  • [c] : 装置文件里面的串行端口设备，例如键盘、鼠标 (一次性读取装置)

接下来的字符：

每三个成一组，

[-] 代表 无该权限。

文件属主

文件的特定所有者，具有该文件的所有权

另外，Linux 中，用户是按组分类的，一个用户属于一个或多个组。

文件所有者意外的用户又可以被分为文件所有者的同组用户和其他用户。

更改文件属性

  1. chgrp : 更改文件属组

  2. chown : 更改文件属主，也可以同时更改文件属组

  3. chmod : 更改文件 9 个属性

    Linux 文件属性有两种设置方式，一种是数字，一种是符号。

    Linux 文件的基本权限有 9 个，分别是 owner/group/others 三种身份各有自己的 read/write/execute 权限。

    数字方法：

    文件权限字符：[-rwxrwxrwx]

    每三个一组，其中每个权限字符有对应的分数：

    • r : 4
    • w : 2
    • x : 1

    比如 rwx = 4 + 2 + 1 = 7; r-x = 4 + 0 + 1 = 5

    所以指令是这样的：

    xyz : 三组文件权限分数。 (777 = rwxrwxrwx; 770 = rwxrwx---)

    -R : 递归变更，即目录下所有文件

    符号类型方法：

    9 个权限分别是 user, group, others 三种身份

    利用 u, g, o 代表三种身份的权限。

    a 代表 all ，即所有身份。

    比如要将文件权限设置成 -rwxt-xr--：chmod u=rwx,g=rx,o=r filename

-a ：相当於 -pdr 的意思，至於 pdr 请参考下列说明；(常用)

-d ：若来源档为连结档的属性(link file)，则复制连结档属性而非文件本身；

-r ：递回持续复制，用於目录的复制行为；(常用)

-s ：复制成为符号连结档 (symbolic link)，亦即『捷径』文件；

-u ：若 destination 比 source 旧才升级 destination ！

Build SSR on VPS

Posted on 2019-02-24

BBR

  1. wget --no-check-certificate https://github.com/teddysun/across/raw/master/bbr.sh && chmod +x bbr.sh && ./bbr.sh

验证是否安装正常

sysctl net.ipv4.tcp_congestion_control

结果为 net.ipv4.tcp_congestion_control = bbr 即成功

SSR

  1. wget --no-check-certificate -O shadowsocks-all.sh https://raw.githubusercontent.com/teddysun/shadowsocks_install/master/shadowsocks-all.sh
  2. chmod +x shadowsocks-all.sh
  3. ./shadowsocks-all.sh 2>&1 | tee shadowsocks-all.log

关于配置选择

  • 选择 ShadowsocksR
  • 端口 随意
  • 推荐加密方式：chacha20-ietf、aes-128-ctr及其它
  • 推荐的最佳协议：auth_sha1_v4_compatible及其它
  • 推荐的最佳混淆：tls1.2_ticket_auth_compatible及其它

带_compatible表示兼容SS原版协议，也就是服务端不用修改协议跟混淆，只要在客户端将协议改为origin ，将混淆改为plain，那么它就成了SS。

这个在利用SSR玩外服游戏时极为重要，因为增加协议与混淆会增加延迟及一些莫名其妙的问题，所有玩外服游戏时建议按以上修改：协议改为origin，混淆改为plain。

注意事项

关于命名

类和ID的命名最好不要以数字开头。

以数字开头的 class 和 ID 在 Mozilla/Firefox 浏览器中不起作用。

关于多重样式的优先级

一般来说优先级如下：

内联样式 > 内部样式 > 外部样式 > 浏览器默认样式

但是，当我们在 \<head> 部分定义外部样式和内部样式时，若将内部样式写在了前面，则多重样式选择时，外部样式会覆盖掉内部样式。
