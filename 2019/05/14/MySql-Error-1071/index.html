MySql Error 1071

Posted on 2019-05-14

When I was trying to move a database from the remote to localhost, I met this error due to the different settings of the local database.

The Error log

It is obvious that the max key length limit caught this error.

So we have two methods to solve it:

  • Change the limit
  • Use a shorter part of the content as index

Change the limit

The Innodb engine of MySql got a parameter named innodb_large_prefix which can change the max key length of the utf8-based database from 767 bytes to 3072.

So we can change the innodb_large_prefix then recreate the database(remember to backup your data first).

There are some methods to change it:

  • Modify the configure file
  • Set some parameters in mysql terminal
